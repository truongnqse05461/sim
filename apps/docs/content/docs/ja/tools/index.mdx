ツールはSimの強力なコンポーネントで、ワークフローが外部サービスと連携したり、データを処理したり、特殊なタスクを実行したりすることを可能にします。これらはさまざまなAPIやサービスへのアクセスを提供することで、エージェントやワークフローの機能を拡張します。

## ツールとは何か？

ツールは特定の機能や外部サービスとの統合を提供する専門的なコンポーネントです。ツールはウェブ検索、データベースとの対話、画像処理、テキストや画像の生成、メッセージングプラットフォームを通じたコミュニケーションなど、さまざまな用途に使用できます。

## ワークフローでのツールの使用方法

Simワークフローでツールを使用する主な方法は2つあります：

<Steps>
  <Step>
    <strong>スタンドアロンブロックとして</strong>: ツールは、その機能に決定論的かつ直接的にアクセスする必要がある場合、キャンバス上の個別のブロックとして追加できます。これにより、ツールがいつどのように呼び出されるかを正確に制御できます。
  </Step>
  <Step>
    <strong>エージェントツールとして</strong>: 「ツールを追加」をクリックして必要なパラメータを設定することで、ツールをエージェントブロックに追加できます。これにより、エージェントはタスクのコンテキストと要件に基づいて、使用するツールを動的に選択できるようになります。
  </Step>
</Steps>

## ツールの設定

各ツールが適切に機能するには、特定の設定が必要です。一般的な設定要素には以下が含まれます：

- **APIキー**: 多くのツールはAPIキーによる認証が必要です
- **接続パラメータ**: エンドポイント、データベース識別子など
- **入力フォーマット**: ツールにデータをどのように構造化するか
- **出力処理**: ツールからの結果をどのように処理するか

## 利用可能なツール

Simは様々な目的のための多様なツールコレクションを提供しています：

- **AI・言語処理**: OpenAI、ElevenLabs、翻訳サービス
- **検索・リサーチ**: Google検索、Tavily、Exa、Perplexity
- **ドキュメント操作**: Googleドキュメント、Googleスプレッドシート、Notion、Confluence
- **メディア処理**: Vision、画像ジェネレーター
- **コミュニケーション**: Slack、WhatsApp、Twilio SMS、Gmail
- **データストレージ**: Pinecone、Supabase、Airtable
- **開発**: GitHub

各ツールには、設定と使用方法に関する詳細な説明が記載された専用のドキュメントページがあります。

## ツールの出力

ツールは通常、ワークフローの後続ブロックで処理できる構造化データを返します。このデータの形式はツールと操作によって異なりますが、一般的に以下を含みます：

- メインコンテンツまたは結果
- 操作に関するメタデータ
- ステータス情報

各ツールの正確な出力形式を理解するには、それぞれのツールの特定のドキュメントを参照してください。

## YAML設定

詳細なYAMLワークフロー設定と構文については、[YAMLワークフローリファレンス](/yaml)ドキュメントを参照してください。これには以下の包括的なガイドが含まれています：

- **ブロック参照構文**: ブロック間でデータを接続および参照する方法
- **ツール設定**: スタンドアロンブロックとエージェント設定の両方でツールを使用する方法
- **環境変数**: APIキーと認証情報の安全な取り扱い
- **完全な例**: 実世界のワークフローパターンと設定

特定のツールパラメータと設定オプションについては、各ツールの個別ドキュメントページを参照してください。
